## Why AlphaLens?

Traditional RAG pipelines treat SEC filings like ordinary text documents. Fixed-size chunking often splits tables, breaks risk factors, and loses the document hierarchy that gives financial disclosures their meaning.

AlphaLens preserves that structure by parsing filings around SEC Items before semantic chunking. Every retrieved answer remains traceable to the underlying filing, making it easier to inspect supporting evidence.

> **Note**
>
> Chunk extraction is validated against the source filing. Final answers are still generated by an LLM and should always be verified using the cited source chunks.

## ✨ Features

### Ingestion Pipeline

| Feature | Benefit |
|---------|---------|
| Structure-aware parsing | Preserves SEC hierarchy instead of fixed token windows |
| Semantic chunking | Reduces broken paragraphs and split tables |
| Text-fidelity validation | Detects accidental changes during processing |
| Metadata enrichment | Improves retrieval with generated headlines and summaries |

### Retrieval Pipeline

| Feature | Benefit |
|---------|---------|
| Query rewriting | Improves retrieval recall |
| Dual retrieval | Searches both original and rewritten queries |
| Merge & deduplicate | Removes redundant candidates |
| Cross-encoder reranking | Keeps only the most relevant context |
| Source-grounded generation | Every answer links back to supporting chunks |

## 🏗 Architecture

AlphaLens consists of two independent pipelines.

### 1. Ingestion Pipeline

| Step | Description |
|------|-------------|
| Ticker Lookup | Resolve company ticker |
| Download | Fetch SEC filing |
| Parse | Extract SEC Items |
| Chunk | Semantic chunking + metadata |
| Embed | Store vectors in Qdrant |

### 2. Retrieval Pipeline

| Step | Description |
|------|-------------|
| Rewrite | Optimize query |
| Retrieve | Dual vector search |
| Merge | Deduplicate candidates |
| Rerank | Jina cross-encoder |
| Generate | Build grounded answer |

## Example Workflow

```text
Company → Download SEC Filing
          ↓
Parse by SEC Item
          ↓
Semantic Chunking
          ↓
Embeddings → Qdrant
          ↓
User Question
          ↓
Rewrite → Retrieve → Rerank
          ↓
Answer + Supporting Sources
```

## ⚙️ Getting Started

```bash
git clone https://github.com/YOUR_USERNAME/AlphaLens.git
cd AlphaLens

python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate

pip install -r requirements.txt
```

Create a `.env`:

```env
GEMINI_API_KEY=
jina=
QDRANT_URL=
QDRANT_API_KEY=
```

Run:

```bash
python app.py
```
